# Product Requirements Document: DSir

## 1. Overview

DSir is an AI-powered programming education platform that takes learners from complete beginners to industry-ready software developers. It uses personalized learning paths, adaptive AI mentoring, intelligent revision, and real-world projects to build genuine mastery.

## 2. Vision

Create one of the highest-quality programming education platforms in the world—capable of serving millions of learners while remaining useful for individuals, teachers, universities, and professionals.

## 3. Target Audience

- Complete beginners with no prior coding experience
- Self-taught developers seeking structured guidance
- University students supplementing formal education
- Professionals upskilling or switching careers
- Teachers and institutions needing adaptive curriculum tools

## 4. Core Objectives

- Teach real, transferable programming skills
- Adapt to each learner's pace, strengths, and weaknesses
- Ensure long-term retention through evidence-based revision
- Prepare learners for real software engineering work
- Support scalable content creation for new technologies and roadmaps

## 5. In-Scope Technologies (Courses)

- Python
- HTML
- CSS
- JavaScript
- TypeScript
- React
- Next.js
- Node.js
- SQL
- Git
- APIs
- Deployment

## 6. In-Scope Career Roadmaps

- Full Stack Developer
- Python Developer

Future roadmaps must be addable without application code changes.

## 7. Key Differentiators

- Concept-based learning rather than chapter-based progression
- AI mentor that understands the learner's full history
- Intelligent revision using spaced repetition and active recall
- Real projects with code review and feedback
- Mastery-based progression instead of completion-based progress

## 8. Success Metrics

- Learner retention after 30, 60, and 90 days
- Mastery score improvement over time
- Time to complete roadmaps
- Project completion and quality scores
- Learner-reported confidence and satisfaction
- Code submission and revision engagement rates

## 9. Constraints

- Must be accessible to learners with disabilities
- Must support internationalization from day one
- Must be deployable via Docker
- Must support provider-agnostic AI integration
- Must be maintainable for at least 10 years

## 10. Non-Goals

- Shallow quizzes or multiple-choice-only assessments
- Fake progress indicators
- Excessive gamification (badges, leaderboards as primary motivators)
- Copy-paste exercises
- Outdated curriculum
